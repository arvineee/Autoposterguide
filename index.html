Introduction

What This Engine Does

posting_engine.py is a fully autonomous news blog autoposter built for Arval Blog News. It runs continuously on Termux (Android), scans Kenya and World RSS feeds, pulls the raw article for context, generates an original 900–1,300-word article using Gemini AI written in a Kenyan journalist voice, attaches an image, and publishes directly to arvalblognews.online via the blog API. Everything — scheduling, duplicate detection, image sourcing, model rotation, log integrity — is handled internally. No manual input is required once the engine is running.

Step 01

Schedule & Quota Gate

1

Is it safe to post right now?

run() → is_posting_window() → check_daily_limits()

Before touching any feed, the engine checks three gates in order. All three must pass or the cycle exits immediately.

#GatePass conditionFail behaviour
1 Daily quota Posts today < DAILY_LIMIT (24) Sleeps until midnight EAT, then resets
2 Posting window Current EAT hour falls within a configured window Logs "Outside windows. Next: Evening in ~Xh" and exits run
3 Window cap Posts in this window today < window's max_posts Logs "Afternoon cap reached (6/6)" and exits run

The number of articles to attempt is calculated as the minimum of three values:

to_post = min(
    POSTS_PER_RUN,      # 2 (or --posts override)
    remaining_quota,    # DAILY_LIMIT − count_today
    window_left         # window.max_posts − window_count_today
)
How the engine tracks time All time checks use datetime.utcnow() + timedelta(hours=3) to get EAT (East Africa Time). The log stores a "date" field as YYYY-MM-DD — when this doesn't match today, the daily count resets to 0 automatically.
Step 02

RSS Feed Scan

2

Collect story candidates from all 16 feeds

collect_candidates() → detect_category() → extract_feed_image()

All 16 feeds (8 Kenya + 8 World) are scanned in sequence with feedparser.parse(). Each entry that passes deduplication becomes a candidate tuple.

What a raw feedparser entry looks like

entry.title
→
"KRA misses revenue target by Ksh 47 billion in first quarter"
entry.link
→
"https://news.google.com/rss/articles/CBMi..."  (redirect, not the real URL yet)
entry.summary
→
'<img src="https://lh3.googleusercontent.com/abc123..." />'
entry.media_content
→
[]  (empty on this entry)

Image extraction order — extract_feed_image()

Three places are checked in priority order:

# Priority 1: media:content tag (common in BBC, Reuters feeds)
if entry.media_content and media_content[0].get("url"):
    return media_content[0]["url"]

# Priority 2: enclosures with image MIME type
for enc in entry.enclosures:
    if enc["type"].startswith("image/"):
        return enc["href"]

# Priority 3: <img src="..."> inside the summary HTML
img_match = re.search(r'<img[^>]+src=["\']([^"\']+)["\']', summary)
if img_match:
    return img_match.group(1)

Category detection — score-based keyword matching

detect_category() scores each category against the lowercased title. The highest-scoring category wins; falls back to Politics if no keyword matches.

# Title: "KRA misses revenue target by Ksh 47 billion in first quarter"
scores = {
    "Politics":  0,
    "Business":  3,  # "kra" + "revenue" + "tax" all match
    "Technology": 0,
    "Health":    0,
    "Sports":    0,
    "World News": 0,
}
# → category = "Business"

Candidate tuple structure

candidate = (
    "KRA misses revenue target by Ksh 47 billion in first quarter",  # title
    "https://news.google.com/rss/articles/CBMi...",                  # link
    "Business",                                                       # category
    "https://lh3.googleusercontent.com/abc123...",                   # rss image (may be blocked)
)

Pool interleaving

After scanning, Kenya and World candidates are shuffled separately then interleaved so each run naturally alternates regions:

interleaved = [kenya[0], world[0], kenya[1], world[1], ...]
Zero-entry warning If a feed returns 0 entries (network issue or feed down), the engine logs a warning and continues to the next feed rather than crashing. Monitor autoposter.log for repeated warnings on the same feed URL.
Step 03

Duplicate Topic Guard

3

Block stories that are too similar to recent posts

is_duplicate_topic() → title_similarity()

Before making any network requests for an article, the candidate title is compared against the last 60 posted titles stored in the log. This prevents publishing two articles about the same story on the same day.

Jaccard similarity calculation

The algorithm strips stop words (including "kenya" / "kenyan") from both titles, computes word sets, then calculates: |intersection| ÷ |union|. Any score ≥ 0.60 is blocked.

# Candidate: "KRA misses revenue target by Ksh 47 billion in first quarter"
# Tokens after stop-word removal:
A = {"kra", "misses", "revenue", "target", "ksh", "47", "billion", "first", "quarter"}

# Recent post: "Treasury orders budget cuts as KRA collects less than expected"
B = {"treasury", "orders", "budget", "cuts", "kra", "collects", "less", "expected"}

intersection = {"kra"}           # 1 shared word
union        = 16 unique words   # A ∪ B
similarity   = 1 / 16 = 0.06    # well below 0.60 → NOT a duplicate
Why "kenya" is excluded from stop words for Kenya stories The default stop list includes "kenya" and "kenyan" — but only for world stories. For stories from Kenya feeds, "kenya" is kept in the token set so that two different Nairobi-centric articles aren't falsely flagged as duplicates of each other purely because both mention Kenya.
Step 04

URL Resolution

4

Follow Google News redirects to the real article

resolve_google_news_url()

Google News RSS entries use opaque redirect URLs like https://news.google.com/rss/articles/CBMi.... The engine must resolve these to get the actual source URL before fetching article content or attributing the source domain.

Resolution strategy (HEAD-first)

# Step 1: try a cheap HEAD request (no body downloaded)
r = requests.head(url, allow_redirects=True, timeout=12)
if "news.google.com" not in r.url:
    return r.url   # resolved!

# Step 2: fallback — some servers reject HEAD, try GET with stream=True
r = requests.get(url, allow_redirects=True, stream=True, timeout=12)
r.close()        # close immediately — we only needed the final URL
if "news.google.com" not in r.url:
    return r.url
Input
→
"https://news.google.com/rss/articles/CBMiABC..."
After HEAD
→
"https://www.standardmedia.co.ke/business/article/2001..."
source_domain
→
"standardmedia.co.ke"  (parsed from netloc, www. stripped)
Non-Google URLs skip this step entirely resolve_google_news_url() checks for "news.google.com" in the URL first. Direct feeds from Nation Africa (nation.africa) or Standard Media (standardmedia.co.ke) return immediately with no HTTP request made.
Step 05

Article Fetch & Context Extraction

5

Download the source article and extract clean text

generate_and_post_article() — article fetch block

A GET request downloads the full article page HTML. This serves two purposes: finding the article's own og:image, and giving Gemini factual grounding (specific numbers, quotes, names) rather than hallucinating.

HTML → plain text (3-stage stripping)

# Stage 1: kill all JavaScript blocks
plain = re.sub(r"<script[^>]*>.*?</script>", " ", raw_html, flags=DOTALL)

# Stage 2: kill all CSS blocks
plain = re.sub(r"<style[^>]*>.*?</style>", " ", plain, flags=DOTALL)

# Stage 3: strip every remaining HTML tag
plain = re.sub(r"<[^>]+>", " ", plain)

# Collapse whitespace and slice to 3,500 chars
plain = re.sub(r"\s+", " ", plain).strip()
article_context = plain[:3500]

Scraped image extraction — scrape_article_image()

Four meta tag patterns are checked in priority order before falling back to the first content <img>:

#Pattern checkedExample value found
1og:image property (content after).../images/kra_hq_nairobi.jpg
2og:image property (content before)(same, attribute order varies)
3twitter:image nameTwitter card image
4twitter:image name (content before)(same)
5First <img src="..."> not matching skip listContent photo inside article body

Skip list filters out tracking pixels and UI chrome:

skip = {"logo", "icon", "pixel", "tracking", "avatar", "1x1", "spinner", "badge", "button"}
What happens if the fetch fails Network errors, paywalls, and 4xx responses are caught silently. The engine logs "Article fetch failed — using headline only" and continues. Gemini will generate the article based on the headline alone, using its own knowledge of current events. The context_block in the prompt changes to: "NOTE: Source article could not be fetched. Draw on your knowledge of current events."
Step 06

Image Pipeline

6

Find, download, and optimise the featured image

download_image() → _resize_image_if_needed() → fetch_wikimedia_image()

Three image sources are tried in order. The first that produces a valid, downloadable image wins. Google-hosted URLs are blocked at every layer.

✕
Layer 1 — RSS feed image Blocked if URL contains: lh3.googleusercontent.com, news.google.com, gstatic.com, googleusercontent.com, googleapis.com/news. Google thumbnails are low-res and legally restricted. RSS image from this example was blocked here.
✓
Layer 2 — Scraped og:image from article page Uses the image the source publisher chose for social sharing. This layer succeeded in the example: standardmedia.co.ke/images/kra_hq_nairobi.jpg
3
Layer 3 — Wikimedia Commons Only reached if Layers 1 and 2 both fail. Extracts 5 meaningful tokens from the title, strips stop words, searches Commons API (list=search in namespace 6). Kenya stories keep "kenya" in the query. Returns thumb URL ≥ 500px wide, JPEG/PNG/WebP only.

Download safeguards — download_image()

# Stream in 16KB chunks, counting bytes
for chunk in r.iter_content(chunk_size=16384):
    total += len(chunk)
    if total > MAX_IMAGE_BYTES:   # 2MB hard cap — protects Termux RAM
        os.unlink(tmp.name)
        return None, None
    tmp.write(chunk)

# Minimum size check — rejects tracking pixels, 1×1 gifs
if os.path.getsize(tmp.name) < 5120:   # < 5KB
    os.unlink(tmp.name)
    return None, None

# MIME type from Content-Type header (not file extension)
ext = _ext_from_content_type(r.headers.get("Content-Type", "image/jpeg"))
# Allowed: .jpg .jpeg .png .webp .gif

# Pillow resize if available
_resize_image_if_needed(tmp.name)     # shrinks to ≤ 1400px wide

filename = f"arval_{int(time.time())}{ext}"  # e.g. arval_1750492449.jpg
Pillow is optional but recommended The engine attempts a silent pip install Pillow at startup if it's not found. Without Pillow, images are uploaded at their original size — which can be 3–5MB for high-res editorial photos. With Pillow, they're resized to 1400×N at quality=85, typically producing files under 200KB. Install with pkg install python-pillow on Termux.
Step 07

Gemini Article Generation

7

Build the prompt and generate the article

generate_article() → generate_and_post_article() — prompt block

The engine assembles a structured prompt, calls Gemini with a rotating model pool and randomised temperature, and detects refusals before accepting the output.

Prompt structure

Blog topic based on this current news event:
HEADLINE: KRA misses revenue target by Ksh 47 billion in first quarter
SOURCE: standardmedia.co.ke

ARTICLE CONTENT FROM SOURCE:
[3,500 chars of stripped article text]

INSTRUCTIONS:
Write a completely original Kenyan news article of 1000–1100 words
using the above as your factual foundation. Go deep — add local
Kenyan context, name real places, quote what ordinary Kenyans are
saying, and take a clear editorial position. The writing must feel
like a tired but passionate Nairobi journalist wrote it at 11pm...
<p style="font-size:12px; color:#999;">Source: standardmedia.co.ke</p>

The word target is randomly chosen from three options each run: "900–1000", "1000–1100", "1100–1300". This prevents all articles being the same length.

Model rotation & retry logic

ModelRoleWhen rotated away
gemini-3.1-flash-litePrimaryRate limit (429) or refusal detected
gemini-3.5-flashFallback 1Same
gemini-3-flashFallback 2Same
gemini-2.5-flash-liteFallback 3After all models cycled twice, gives up

On quota hit, exponential backoff fires: min(2^attempts + random(1,4), 60) seconds.

AI-detection countermeasures

Three mechanisms work together to make the output feel human-written:

Temperature Randomised between 0.75 and 0.95 per call. Higher = more varied, natural-sounding output.

Banned words The system prompt explicitly prohibits 20+ AI giveaway phrases ("Furthermore", "Delve into", "It is worth noting", "Landscape", etc.).

Voice rules Swahili phrases mid-sentence, first-person moments ("I called the ministry. Nobody picked up."), Nairobi estate name-drops, and punchy one-line paragraphs are all specified in the system prompt.

Refusal detection

REFUSAL_SIGNALS = [
    "i can't", "i cannot", "i'm unable", "as an ai",
    "i don't have the ability", "i must decline", ...
]
# Checked against the first 500 chars of the response
# If matched → rotate model and retry, don't post
Step 08

Content Parsing & Validation

8

Extract every component from the raw Gemini response

generate_and_post_article() — parse block

The raw Gemini response is a single string containing HTML, a meta description comment, and a Quote Card line. Each component is extracted with a dedicated regex.

What Gemini is expected to return

```html
<h1 style="...">KRA Ksh 47 Billion Short: Who Is Keeping The Books?</h1>
<p style="...">Six hundred and thirty billion. That was the number...</p>
...
<footer style="...">
  ...
  <p style="font-size:12px; color:#999;">Source: standardmedia.co.ke</p>
</footer>
<!--META-DESCRIPTION:KRA collected Ksh 583 billion against a Ksh 630 billion
target in Q1, raising questions about fiscal management in Kenya.-->
```
Quote Card Text: Forty-seven billion missing and nobody has been arrested.

Extraction sequence

WhatRegexFallback
HTML block ```html\s*(.*?)\s*``` Bare ``` block, then raw text
Word count Strip all tags → split → count Discard if < 400 words
Meta description <!--META-DESCRIPTION:(.*?)--> Empty string (no meta tag posted)
Quote Card Quote Card Text:\s*(.+?)(?:\n|$) None (not logged)
Article title <h1[^>]*>(.*?)</h1> "Exposed: The Real Story Behind {source_title}"

After extraction, the <h1> tag is stripped from content_html — the blog CMS renders the title separately above the article body, so including it in the content would duplicate it.

The <!--META-DESCRIPTION:...--> comment is also stripped from content_html before posting, since it's extracted separately and sent as its own API field.

Two reasons an article is discarded here (1) Word count below MIN_WORD_COUNT (400) — usually means Gemini returned a partial or truncated response. (2) Refusal signals found in the stripped plain text — means the model refused inside an HTML wrapper, which _contains_refusal() catches by checking the plain text too, not just the raw response.
Step 09

Blog API Submission

9

POST the article and image to arvalblognews.online

generate_and_post_article() — API block

A human-like delay (3–9 seconds random) fires before the request. Then the API is called with up to 3 retries on server errors.

Payload fields

FieldValue (example)Type
title"KRA Ksh 47 Billion Short: Who Is Keeping The Books?"Form field
contentFull HTML body (h1 stripped, meta comment stripped)Form field
category_name"Business"Form field
status"published"Form field
meta_description"KRA collected Ksh 583 billion against..."Form field
imagearval_1750492449.jpg (binary)Multipart file

Two request paths

# Path A: image downloaded successfully — multipart upload
with open(image_file_path, "rb") as img_f:
    files = {"image": (image_filename, img_f, mime_type)}
    res = requests.post(ARVAL_API_URL,
        headers={"X-API-Key": ARVAL_API_KEY},
        data=payload, files=files, timeout=50)

# Path B: image failed — send URL instead (blog fetches it server-side)
payload["featured_image_url"] = featured_image_url
res = requests.post(ARVAL_API_URL,
    headers={"X-API-Key": ARVAL_API_KEY},
    data=payload, timeout=35)

Retry logic on 5xx

for attempt in range(1, 4):    # up to 3 attempts
    ...
    if res.status_code == 201:
        break                  # success
    elif res.status_code >= 500:
        time.sleep(5 * attempt)  # 5s, 10s, 15s
    else:
        break                  # 4xx — no point retrying

Canonical URL construction (on 201)

cat_slug       = "business"   # category_name.lower().replace(" ", "-")
canonical_slug = slugify(title) # mirrors slugify() in routes.py exactly
               = "kra-ksh-47-billion-short-who-is-keeping-the-books"
live_url       = f"https://www.arvalblognews.online/news/{cat_slug}/{canonical_slug}"
Temp image cleanup always runs The /tmp/tmpXk8z2q.jpg file is deleted in a finally-style block after the API call, whether the request succeeded or failed. Temp files never accumulate on device storage.
Step 10

Post-Publish Actions

10

Notify, index, and commit to log

send_termux_notification() → submit_to_indexnow() → commit_to_log() → save_log()

Three things fire in fixed order after a 201 response. All are non-blocking — failures are logged as warnings and never prevent the engine from continuing.

1. Termux notification

termux-notification \
  --title  "✅ Arval: KRA Ksh 47 Billion Short..." \
  --content "Business | 1063w | /news/business/kra-ksh-47..." \
  --id      1000 \
  --priority high \
  --sound \
  --vibrate 200,100,200

Notification IDs increment with each post (_notif_id), so articles don't overwrite each other on the notification shade. If termux-notification isn't installed, the error is caught silently.

2. IndexNow submission

POST https://api.indexnow.org/indexnow
{
  "host":        "www.arvalblognews.online",
  "key":         "arvalblognews",
  "keyLocation": "https://www.arvalblognews.online/static/arvalblognews.txt",
  "urlList":     ["https://www.arvalblognews.online/news/business/kra-ksh-47..."]
}

Notifies Bing and Yandex for near-immediate crawl. IndexNow failures are non-fatal — the post is already live regardless.

3. Atomic log write — commit_to_log() → save_log()

Every successful post updates seven fields in the JSON log:

FieldUpdate
countIncremented by 1
dateSet to today's date (YYYY-MM-DD)
processed_urlsSource Google News URL appended (prevents reprocessing)
posted_titlesFinal article title appended (used for duplicate detection)
window_counts"2026-06-21_Morning" counter incremented
quote_cardsQuote Card text + ISO timestamp appended
run_statsObject with ts, title, words, model, image flag, window appended

The write is atomic — JSON is serialised to .json.tmp first, then os.replace() renames it. The previous log is backed up to .json.bak before every write. If the main log is corrupted on next load, the backup is tried automatically.

processed_urls is never cleared between days The daily count resets at midnight, but processed_urls persists across days. This means the engine will never republish the same source article on a different day, even if it becomes trending again. The list is trimmed to a maximum of 600 entries (oldest removed first) to prevent the log file from growing indefinitely.

CLI Flags

All flags are parsed at startup via argparse and stored in ARGS.

FlagTypeWhat it does
--dry-run Boolean Generates content and logs everything, but does NOT call the blog API, send notifications, or submit to IndexNow. URL is still committed to the log so it isn't reprocessed.
--once Boolean Runs exactly one posting cycle then exits. Useful for cron job integration or manual triggers.
--posts N Integer Overrides POSTS_PER_RUN for this session only. --posts 5 will attempt up to 5 articles per cycle.
--test-image URL String Downloads the given image URL through the full pipeline (stream cap, MIME check, Pillow resize) and reports the result. Engine exits immediately after. Useful for debugging image sources.

Common usage patterns

# Test everything without posting
python posting_engine.py --dry-run --once

# Catch up — post 4 articles right now then exit
python posting_engine.py --once --posts 4

# Debug why an image from a Standard Media article isn't working
python posting_engine.py --test-image https://www.standardmedia.co.ke/images/story.jpg

# Normal continuous run (default)
python posting_engine.py

Troubleshooting

SymptomLikely causeFix
Engine exits immediately without scanning Outside posting windows or daily limit full Check autoposter.log for "Outside windows" or "Daily quota full"
"Recovered from backup log file" Previous write was interrupted mid-file Normal — the backup recovery is working. Delete both log files to reset from scratch if needed.
All articles posting without images Pillow not installed; all image layers failing Run pip install Pillow; test a specific URL with --test-image
"Gemini refusal detected" on every model Story topic triggering safety filter Normal — engine skips the story and moves to the next candidate
"Output too short: N words" Gemini returned a partial response or very short refusal Raise MIN_WORD_COUNT to catch more edge cases, or lower it if legitimate short articles are being dropped
API 401 on every post ARVAL_API_KEY wrong or not loaded Check .env file exists in the same directory; verify the key value
Duplicate articles appearing on the blog posted_titles list cleared, or similarity threshold too high Lower SIMILARITY_THRESHOLD from 0.60 toward 0.45
"Feed returned 0 entries" Feed URL dead or rate-limited Check the feed URL in a browser; remove or replace it from KENYA_FEEDS / WORLD_FEEDS
Posts appearing in wrong category Title doesn't match any keyword in CATEGORY_KEYWORDS Add relevant keywords to the appropriate category dict; all falls back to "Politics"
